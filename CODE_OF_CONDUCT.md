# Code de conduite

Ce projet correspond à mon site web personnel. Les échanges restent rares mais doivent toujours se dérouler dans un climat respectueux, ouvert et inclusif.

## Notre engagement

Je m'engage à faire de ce dépôt et de ses espaces associés (issues, discussions, courriels) un lieu exempt de harcèlement et accueillant pour tous, indépendamment du genre, de l'origine, de l'orientation, des capacités ou de l'expérience.

## Standards attendus

Comportements encouragés :
- Faire preuve d'empathie et de respect dans les échanges.
- Partager des retours constructifs et concrets.
- Aider les nouveaux arrivants et valoriser leurs contributions.

Comportements inacceptables :
- Harcèlement, propos haineux ou discriminatoires.
- Attaques personnelles, trolling, insinuations agressives.
- Partage d'informations privées sans consentement.
- Publication de contenu malveillant ou trompeur.

## Responsabilités

Je me réserve le droit de modérer les issues, commentaires et contributions qui ne respecteraient pas ces principes. Les mesures possibles incluent l'avertissement, la fermeture d'une discussion ou le refus/bannissement de contributions futures.

## Portée

Ce code s'applique dans le dépôt GitHub, les discussions liées au site et tout espace où quelqu'un représente ce projet.

## Signalement

Si vous subissez ou constatez un comportement inapproprié, signalez-le par courriel à : wrichard@live.fr. Je m'efforcerai de répondre sous 72h et de traiter la demande avec confidentialité.

## Processus et conséquences

1. Accusé de réception du signalement.
2. Revue des faits et échange avec les personnes concernées.
3. Mesure proportionnée : rappel à l'ordre, avertissement, fermeture de l'échange ou exclusion des contributions.

### Checklist de mise en place

Avant de publier ce Code de Conduite :

- [ ] Remplacer tous les `[PLACEHOLDERS]` par les informations réelles
- [ ] Créer l'adresse email de contact dédiée
- [ ] Désigner au moins 2 mainteneurs responsables de l'application
- [ ] Former l'équipe de modération aux meilleures pratiques
- [ ] Ajouter un lien vers le Code de Conduite dans le README.md
- [ ] Mentionner le Code de Conduite dans CONTRIBUTING.md
- [ ] Configurer les templates GitHub/GitLab pour référencer le Code
- [ ] Communiquer l'adoption du Code à la communauté existante
- [ ] Préparer un document interne de procédures de modération (confidentiel)

### Outils recommandés

- **GitHub Community Health Files** : Ajouter automatiquement le Code aux repos de votre organisation
- **Discord AutoMod** : Configurer des filtres automatiques basés sur vos standards
- **Formulaire de signalement** : Google Forms ou Typeform pour rapports anonymes
- **Base de données d'incidents** : Notion ou Airtable pour tracking (accès restreint)

### Formation des modérateurs

Sujets à couvrir :
1. Biais implicites et reconnaissance
2. Écoute active et empathie
3. Désescalade de conflits
4. Prise de décision sous pression
5. Limites personnelles et prévention du burnout

### Révision périodique

Planifier une révision du Code tous les 12 mois :
- Évaluer l'efficacité (nombre d'incidents, résolutions satisfaisantes)
- Mettre à jour selon les évolutions de la communauté
- Intégrer les retours d'expérience
- Aligner avec les nouvelles versions du Contributor Covenant
