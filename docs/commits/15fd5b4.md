# chore(format): configurer le formatage du code

## Objectif

Standardiser le formatage du code sur l'ensemble du projet pour maintenir une cohérence et améliorer la lisibilité. Tous les développeurs utiliseront les mêmes règles de formatage automatiquement.

## Références

- [EditorConfig](https://editorconfig.org/)
- [Prettier](https://prettier.io/)
- [Prettier + ESLint](https://prettier.io/docs/en/integrating-with-linters.html)
- [Guide de style JavaScript](https://google.github.io/styleguide/jsguide.html)

## Problème résolu

Sans configuration de formatage uniforme :

- Indentation inconsistante (2 espaces vs 4 espaces vs tabs)
- Lignes trop longues difficiles à lire
- Formatage différent selon les éditeurs
- Conflits Git inutiles dus aux différences de formatage

## Fichiers créés/modifiés

### `.editorconfig`

Configuration universelle pour tous les éditeurs (VS Code, IntelliJ, Sublime...)

### `.prettierrc`

Configuration Prettier pour le formatage automatique JavaScript/JSON/Markdown

### `.prettierignore`

Fichiers à exclure du formatage automatique Prettier

### `package.json` (mise à jour)

Ajout des dépendances et scripts Prettier

- `prettier` : Outil de formatage
- `eslint-config-prettier` : Désactive les règles ESLint conflictuelles avec Prettier
- Script `format` : Formate tous les fichiers

### `.eslintrc.json` (mise à jour)

Intégration Prettier avec ESLint pour éviter les conflits entre ESLint et Prettier

```bash
# Linter (inclut maintenant Prettier)
npm run lint
```
