# feat(pages): créer la structure de base de l'application Next.js

## Objectif

Mettre en place la structure minimale de l'application Next.js avec le Pages Router pour permettre le démarrage du serveur de développement et afficher une page d'accueil fonctionnelle.

## Structure créée

```text
wylow2ricard0.github.io/
├── pages/
│   └── index.js          ← Page d'accueil (/)
├── .eslintrc.json
├── docs/commits/
    ├── 475a872.md
    └── 948532e.md
└── ...
```

## Notes importantes

1. **Pages Router vs App Router** : On utilise Pages Router (plus simple pour débuter)
2. **index.js obligatoire** : C'est la convention pour la route `/`
3. **Hot reload** : Les modifications sont visibles sans redémarrage

## Références

- [Next.js Pages Router](https://nextjs.org/docs/pages)
- [Pages structure](https://nextjs.org/docs/pages/building-your-application/routing/pages-and-layouts)

## Problème résolu

Erreur rencontrée :

Next.js nécessite obligatoirement un dossier `pages/` (Pages Router) ou `app/` (App Router) pour fonctionner.

## Fichiers

### `pages/index.js`

Page d'accueil principale accessible à la route `/`

### `.eslintrc.json`

Configuration ESLint pour Next.js

### Tester les scripts

1. `npm run dev` : Serveur de développement

   ```shell

   ```

   Next.js nécessite obligatoirement un dossier `pages/` (Pages Router) ou `app/` (App Router) pour fonctionner.

1. `npm run lint` Vérification ESLint réussie
1. `npm run build` Optimisation des assets
