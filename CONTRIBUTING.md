
# Contribution

Ce dépôt héberge mon site personnel. Les contributions restent limitées mais les suggestions sont appréciées pour corriger des fautes, des liens cassés ou proposer des améliorations légères.

## Proposer une idée ou signaler un souci

- Ouvrez une Issue (lien "Issues" sur GitHub) en décrivant le problème ou la suggestion.
- Merci d'indiquer une URL concernée si le souci est sur la version publiée.

## Pull Requests (cas par cas)

Les PR sont acceptées principalement pour :
- corrections typographiques ou liens morts ;
- petites améliorations de lisibilité ;
- mises à jour mineures de configuration.

Workflow conseillé :
1) Forker le dépôt, puis `git checkout -b fix/description-courte`.
2) Faire les modifications ciblées et vérifier qu'elles s'affichent correctement en local (`public/` + prévisualisation Docker si nécessaire).
3) `git commit -m "fix: corrige lien cassé"` puis `git push` vers votre fork.
4) Ouvrir une PR avec un résumé clair et, si possible, une capture ou lien de prévisualisation.

## Qualité attendue

- Pas de fichiers générés ou de dépendances non utilisées dans les PR.
- Pas de secrets, jetons ou données personnelles dans le dépôt.
- Orthographe et français soignés ; titres courts ; liens testés.
- Respect du [Code de conduite](CODE_OF_CONDUCT.md).
