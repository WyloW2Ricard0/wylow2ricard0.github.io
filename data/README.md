# data/

Dossier réservé aux contenus et ressources statiques du site personnel.

Utilisations typiques :
- Images optimisées (préférer webp/avif) avant copie dans `public/`.
- Fichiers JSON/YAML pour lister projets, liens ou notes.
- Brouillons Markdown avant publication.

Bonnes pratiques :
- Ne pas committer de secrets ou données personnelles.
- Garder les fichiers lourds hors du dépôt (utiliser un lien externe si besoin).
- Nommer clairement les ressources (ex. `projets.json`, `avatar.webp`).
